Abstract
The challenging task of earthquake (EQ) prediction has recently gained significant atten-
tion, particularly with machine learning techniques. Geomagnetic field analysis has yielded 
promising results in identifying EQ precursors. However, the complexity of the data has 
made it difficult to create an accurate model for EQ prediction using this method. This 
study presents an automated machine learning (AutoML) approach capable of handling the 
complexity of geomagnetic data and selecting the most suitable model. A dataset contain-
ing 50 years of geomagnetic field data was collected, of which the measurements were 
taken in close proximity to M5.0+ EQs. The study demonstrated that sampling techniques 
can overcome the problem of an imbalanced dataset from EQ events. Through statistical 
analysis, important features were extracted and a multi-class classification model using 
geomagnetic data was created. The extracted features were the input for AutoML, an auto-
matic algorithm selection that was measured by Bayesian Optimization algorithm to select 
the best performance model. The results indicate that the neural network model outper-
formed eight other classifiers, achieved an accuracy of 81.19%, F1-score of 80.51%, and 
a Matthews Correlation Coefficient (MCC) of 77.49%. It is concluded that the neural net-
work multi-class classification model is capable of providing solutions to the challenges 
faced when using geomagnetic data for EQ prediction.

to EQs, which could emerge as a result of changes in Earth’s crust and rock stress before 
an EQ. A finding by (Han et al. 2020) through a stochastic test showed that pre-EQ geo-

However, the relationship between geomagnetic field data and EQs has not been fully 
understood and developing accurate models for EQ prediction using geomagnetic data 
remains a challenging task. This is due to the complexity of geomagnetic data, which can 
contain non-linear patterns and relationships that are difficult to capture using mathemati-
cal, computational, geological and even traditional machine learning methods (Balasis 
et al. 2016). Furthermore, unsupervised machine learning has the capability to swiftly ana-
lyze seismicity catalogues, and the technique can make more thorough representations of 
seismic activity visible, allowing hidden patterns (to be detected) and seismic events to be 
more accurately forecast (Beroza et al. 2021). Deep learning techniques enhance under-
standing of the temporal and spatial evolution of EQ sequences that can efficiently repro-
cess complex EQ data. These algorithms can efficiently and accurately spot intricate and 
subtle patterns in seismic activity, aiding the understanding of the physics underpinning 
EQs (Cianetti et  al. 2021). Deep learning approaches show great promise for identify-
ing additional low-frequency EQ sources which could improve our understanding of EQs 
(Thomas et al. 2021).
Automated machine learning (AutoML) is a framework that automates several machine 
learning tasks including model training and hyperparameter tuning. It can perform data 
exploration and preprocessing, feature extraction and selection, model selection, and 
hyperparameter tuning automatically (Feurer et al. 2015). It can also handle large volumes 
of complex geomagnetic data and can find the best model among various machine learning 
algorithms. AutoML has been found to improve the performance of models and reduce the 
time required to develop them (Hutter et al. 2019). A number of studies have used various 
conventional machine learning algorithms such as support vector machines (SVMs) (Akyol 
et al. 2020), ensemble (Asim et al. 2018), and deep learning algorithms such as artificial 
neural networks (ANNs) to predict EQs (Adeli and Panakkat 2009; DeVries et al. 2018).
Being able to predict the magnitude of impending EQs is imperative (Ku et al. 2021); 
therefore, a multi-class classification approach was adopted in this study. However, imbal-
anced data across classes is a common problem in machine learning, where in this study, 
the majority of the data corresponds to non-EQ events and low magnitude events (Chelidze 
et al. 2022). This data imbalance can lead to model performance issues because machine 
learning algorithms may be biased toward the majority class (moderate and strong EQs) 
and not perform well on the minority class (great EQ). Imbalanced data can have a sig-
nificant impact on the accuracy of machine learning algorithms and for this reason, proper 
techniques are necessary for addressing the problem (He and Garcia 2009). Resampling 
techniques such as Random Undersampling, Synthetic Minority Oversampling Technique, 
and ratio sampling have been proposed to mitigate this issue (Chawla et al. 2002).These 
methods aim to balance the distribution of the data between the different classes, reducing 
the bias toward the majority class.
In this study, the model was trained on a large geomagnetic field dataset and a com-
bination of feature extraction with multi-class classification through AutoML techniques 
to predict EQs. The performance of the model was then evaluated using a combination 
of suitable classification metrics to understand non-linear and imbalanced EQ events. Per-
formance metrics such as precision, recall, F1-Score and Matthew Correlation Coefficient 
(MCC) were then used to evaluate the model’s performance.

In summary, this research aimed to contribute to the field of EQ prediction by develop-
ing a multi-class classification AutoML model that can accurately predict EQs using geo-
magnetic field data so that better understanding of the relationship between geomagnetic 
field data and EQs could be attained. The study sought to explore the challenges associated 
with using geomagnetic data for EQ prediction and propose solutions to overcome these 
challenges.
2  Data and methodology
In this study, geomagnetic data and EQ events that occurred between 1970 to 2020 were 
used, covering a wide range of global EQ events of different magnitudes that were classi-
fied into different classes. A vast dataset of 1-minute geomagnetic data was obtained from 
the SuperMAG database (www.​super​mag.​jhuapl.​edu) which collects and consolidates 
geomagnetic data from a network of 130 magnetometers stationed throughout the world. 
The geomagnetic data were then collated with the EQ data which were obtained from the 
United States Geological Survey (USGS) (www.​earth​quake.​usgs.​gov).
A series of data processing procedures were applied to the EQ and the geomagnetic 
field datasets. These procedures were applied to filter and decluster both datasets, preparing 
them for further analysis and for developing the AutoML model. In order to obtain accu-
rate findings for studies involving Earth’s magnetic field, data filtering is performed based 
on global geomagnetic indices, such as the Ap and Dst indices. The Ap index measures 
Earth’s magnetic field strength; values below 27 indicate quiet conditions. The Dst index 
measures the ring current strength, using a cutoff of − 30 to exclude severe disturbances. 
Both Ap and Dst thresholds are effective in filtering out days with disturbed magnetic field 
conditions, providing a reliable dataset for further analysis (Xu et al. 2013). A comprehen-
sive study database was established using the geomagnetic field data measurements and 
the EQ magnitude which were defined to meet specific criteria. The criteria required that 
the hypocentral location (the location where the EQ originated) should not be more than 
200 km in depth, and the magnitude should not be lower than M5.0 (Hayakawa 2015). In 
this study, only recorded occurrences of mainshock EQs were considered together with 
geomagnetic field data that fell within a seven day window prior to the EQ event up to 
seven days after the actual time of the EQ event.
Statistical features help to capture important patterns and trends in the data that might 
not be apparent from the raw features and this helps to improve the performance of 
machine learning algorithms by providing additional information about the structure of the 
data (Cawley and Talbot 2010). In this study, the statistical approach successfully gener-
ated 120 features from the raw geomagnetic data that were calculated based on various 
parameters including mean, median, minimum, maximum, variance, standard deviation, 
kurtosis, and sum outliers. In this study, EQ events were categorized into multiple intensity 
classes to enable multi-class classification modeling.The full statistical details are provided 
in Supplementary Material 1. The classification scheme was derived from the Modified 
Mercalli Intensity (MMI) scale and mapped according to EQ magnitude ranges. Specifi-
cally, events were grouped as follows: class VI (M5.0−5.5), class VII (M5.5−6.0), class 
VIII (M6.0−6.5), class IX (M6.5−7.0), class X (M7.0−7.5), class XI (M7.5−8.0), and class 
XII (greater than M8.0). This refined classification approach allows for more granular dis-
crimination of EQ intensity and provides a suitable framework for multi-class classification 
using machine learning techniques (Wood and Neumann 1931).

Random Undersampling, Synthetic Minority Oversampling Technique (SMOTE) and 
ratio sampling were subsequently applied to balance the data distribution between the dif-
ferent classes. Next, data were balanced proportionally to each earthquake magnitude class. 
SMOTE is a method of oversampling the minority class (Major and Great EQ events) by 
creating synthetic examples where the difference between the selected example and each 
of its k-nearest neighbors are taken and multiplied by a random number between 0 and 1.
T-Distributed Stochastic Neighbor Embedding (t-SNE) was used to identify feature 
level patterns and interrelationships within the dataset. t-SNE can help identify clusters 
or groups of data points that are similar to each other identifying patterns and trends in 
the data, and providing better understanding the underlying structure of the data (van der 
Maaten and Hinton 2008).
Results of data visualization from t-SNE presented in as shown in Fig. 1 show that the 
data distribution is skewed toward the majority class, with the minority class being under-
represented. This can be observed in the t-SNE plot in Fig. 2, where the data were evenly 
distributed. SMOTE and Random Undersampling can help improve the performance of 
machine learning models by ensuring that the data distribution is more balanced. This can 
be observed in the t-SNE plot, where the minority class is no longer underrepresented and 
is now better separated from the majority class. It could be observed that the distribution 
of data points within each class will also be more even, allowing for easier identification 
of patterns and relationships within each class. When the data distribution is balanced, it 
means that the number of instances in each class is roughly equal and not heavily skewed 
toward one class.
The balanced dataset was trained using AutoML that included automatic training and 
tuning within a specific set of parameters. The data were split into 70% training and 30% 
test sets to run the AutoML model. The model was trained using a combination of various 
machine learning algorithms such as discriminant analysis, ensemble, kernel classification, 
k-nearest neighbor, linear classification, naive bayes, neural network classifier, support vec-
tor machine classifier, and binary decision tree classification that functioned as the Learner.

The best performing model was selected based on its performance on the training data by 
Bayesian optimization. Bayesian optimization balances exploration and exploitation of the 
model space, allowing for efficient and effective optimization of hyperparameters, which 
are the parameters that control the learning process of a model using a combination of grid 
search and random search. Bayesian optimization uses a probabilistic model of the under-
lying function to guide the search for the optimum set of hyperparameters. The model is 
updated after each iteration, allowing for an informed decision about the next set of hyper-
parameters to evaluate. The iterations were optimized from 5-fold cross validation, where a 
total of 850 iterations were used for the raw and sample dataset. Cross validation is a com-
mon technique used to assess the performance of machine learning models. In k-fold cross-
validation, the dataset is divided into k subsets, or "folds". The model is trained on k-1 of 
these folds, and the remaining fold is used for testing. This process is repeated k times, 
with each fold being used for testing exactly once. One way to use cross validation to deter-
mine the number of iterations in a machine learning model is to perform cross validation 
with different numbers of iterations and choose the number that gives the best performance 
on the validation data (Yadav and Shukla 2016).

metrics that could account for the non linear and imbalanced data. Standard performance 
metrics accuracy precision, recall and F1-Score were used. The following are the multi-

3  Results and discussion
The results of the study showed that the Sample (shuffled) model achieved the best per-
formance in terms of accuracy, precision, recall, F1-score, and MCC, making it the most 
effective model for EQ prediction. The Raw model was trained using the original imbal-
anced raw dataset and the Sample model was trained using a balanced sample dataset cre-
ated through synthetic sampling. The Sample (shuffled) model demonstrated significant 
improvement in predicting higher intensity EQ classes due to the application of data bal-
ancing techniques using synthetic sampling.
Bayesian optimization was applied to identify the best performing algorithm for each 
dataset. This method optimizes the objective function by iteratively updating probabilities 
to find optimal parameters. The Sample (shuffled) model identified a neural network as 
the best learner through Bayesian optimization. The neural network achieved optimal per-
formance using the ReLU activation function, no standardization, a regularization param-
eter 휆= 8.0973 × 10−5 , and a two layer architecture with hidden layer sizes of 279 and 63.

The neural network model trained on the Sample (balanced) dataset demonstrated 
strong capability in learning complex patterns and generalizing to unseen earthquake 
events. Its use of the ReLU activation function enhanced computational efficiency and 
supported faster convergence during training. Moreover, the standardized lambda regu-
larization parameter contributed to mitigating overfitting by reducing sensitivity to the 
scale of input features and stabilizing the optimization process (Srivastava et al. 2014). 
Hyperparameters, including layer size, activation function, and regularization strength, 
were optimally selected using Bayesian optimization. This enabled the network to achieve 
high predictive accuracy for multi-class earthquake intensity classification based on Mer-
calli scale labels. In contrast, the ensemble model optimized through Bagging exhibited 
superior performance on the Raw (imbalanced) dataset. Bagging improved robustness by 
aggregating predictions from multiple decision trees trained on different bootstrapped data 
subsets, effectively reducing individual classifier biases. This approach proved particularly 
beneficial under class imbalance, where the ensemble model generalized better than indi-
vidual learners. Accordingly, while the neural network outperformed the ensemble method 
on the balanced dataset due to its capacity for capturing non linear relationships in evenly 
distributed data, the ensemble approach was more effective in handling skewed class dis-
tributions. These results underscore the importance of aligning model selection with data 
characteristics and highlight the utility of automated hyperparameter tuning for optimizing 
earthquake prediction performance.
The Sample (shuffled) model demonstrated the most reliable performance based on met-
rics and validation loss as shown in Fig. 3. The confusion matrix demonstrated that the 
Sample (shuffled) model was able to predict EQ events across all intensity classes, from 
moderate to great EQs, with minimal bias toward any specific class. In contrast, the Raw 
(shuffled) model, as shown in Fig. 4, exhibited strong bias toward the majority class (VI), 
and failed to predict higher intensity events (classes IX to XII), resulting in sparse or zero 
predictions for those categories.
The confusion matrices revealed that the Sample (shuffled) model achieved broader 
coverage and superior generalization across all seismic intensity classes compared to the 
Raw (shuffled) model. This improvement is attributed to the balanced dataset obtained via 
synthetic resampling, which enhanced predictive performance, particularly for underrepre-
sented high magnitude earthquake classes.
Although the Raw (shuffled) model reached an overall accuracy of 69.47%, it did so 
by disproportionately predicting the dominant class. This resulted in zero F1-score and 
MCC due to a complete lack of true positive predictions in minority classes-rendering 
the model ineffective for practical multi-class earthquake classification. In contrast, the

sets but also temporal integrity to achieve reliable forecasting performance.

4  Conclusion
In conclusion, class balancing through SMOTE and Random Undersampling significantly 
improved the performance of a multi-class classification model for EQ intensity predic-
tion using geomagnetic field data. The AutoML framework, optimized through Bayesian 
Optimization (BO), consistently selected a neural network model for the balanced (Sam-
ple) dataset, achieving 81.19% accuracy, 80.51% F1-score, and 77.49% MCC. In contrast, 
the Raw (shuffled) model trained on imbalanced data yielded lower accuracy (69.47%) 
and failed to detect minority classes. However, analysis using non-shuffled, time ordered 
data revealed that the Sample (non-shuffled) model’s performance dropped drastically-
accuracy fell to 4.76% with all key metrics collapsing-indicating that synthetic balancing 
disrupted the temporal structure necessary for generalization. The Raw model maintained 
moderate performance under the same condition, highlighting the need for balancing data 
without compromising temporal integrity. These findings highlight that AutoML com-
bined with proper feature engineering and validation strategies offers a promising path for
